Epics and User Stories

Building your project - like a video game - Each epic is a level, and anything else that doesn’t work toward completing that epic (finishing that level) is a side quest. Focus on beating the game first, then go back and complete the side quests.

Writing Epics

Focus on the “why”. Why is this epic important? What problem is it solving? What important feature is it adding? How is it adding value to the user? If your epic’s have titles like “set up backend” then you need to shift your mindset up a level. Why do you need the backend? For convenience, so I can view my data on multiple devices. There’s your epic. Backend is a checkpoint in completing that epic. Think of the epic like a level in the video game.

Focus on Completing Your Personal Project: Video Game Analogy

Think of your personal project like playing a video game, where each major task or "epic" is a level. You have to beat these levels to win the game. Distractions that don't contribute to beating that level are just side quests. Stay focused on beating the game before getting lost in side quests.

Crafting Epics

When it comes to writing your epics, it's all about 'why'. Why does this epic matter? What problem is it solving? What feature is it adding? How is it making things better for the user? If your epics are titled something like "set up backend," then you need to take a step back and try to view the bigger picture.

Ask yourself: “why do I need a backend?” Maybe it's so you can check your data from any device. There's your epic! Setting up the backend is just a checkpoint (story) in finishing this level (epic).

The way to look at it is, each epic should answer "why am I building this?". Each story within the epic should answer "what am I building". And each task in a story should answer "how am I building this".

User Stories Example

This is an example from one of my mentees who is building a travel app.

Each top level bullet represents an “Epic”, and each sub-bullet underneath is a “story”

  • I need to be able to create a general wishlist.
    • When I am on the “plan trip Overview page” for a given trip, I should see a “+” button at the top of the screen to add to my wishlist.
    • When I click on the “+” button, then I should be taken to a new page with a search bar.
    • When I type into the search bar on the “add attraction page”, then I should start to see results appear for the city that I am currently planning for
    • When I click an attraction result, then I should go back to the plan trip page, with the clicked result added to the list of attractions on that page.
    • Attraction “overview view”:
      • basic quick description
      • how long I plan to spend there (blank at start)
    • If I want to be able to add an attraction from a map view, then…
  • I need to be able to define specific details about each attraction and how I plan to visit them
    • Attraction “detail view”
      • hours by day
      • more detailed description
      • location (with a map view)
        • need to integrate with Google Maps API
      • images of the place
        • Where am I going to source the images?
      • how long I plan to spend there (editable)
      • what kind of attraction is it (food, fun, etc.)
  • I need to group saved places on the wishlist by geographic proximity and length of time spent at each place (with number of groups being number of days of vacation, and with each group containing 0-2 food places, 0-4 snack places, etc.).
    • When I am trying to group attractions by day on itinerary, I need to be able to limit the amount that I can add to a group
      • This story could just be hardcoding group limits (maybe no more than 8 hours in a day if you add up planned time for all attractions in the group)
      • Or, it can be configurable in the UI, which would be an additional story
    • When I go to the itinerary page of the trip I am planning, then I should see a breakdown of each individual day of my planned trip
    • By default, when I am on the itinerary page, all of the attractions should appear at the bottom in an “unscheduled” section
    • I should be able to drag and drop attractions into each day that has enough availability.
    • When I click on the “schedule for me” button, then a modal should appear with a loading spinner while it creates my trip plan
    • When the trip plan is complete, then I should see an itinerary in the modal, with an option to accept or reject the itinerary
    • When I choose to accept the AI itinerary, then the AI itinerary becomes my itinerary and the modal closes, and I see the itinerary.
  • I need an AI schedule for me algorithm
    • Need an API that can take a list of attractions, and a list of dates, and populate them into the dates
    • This should take into account transportation time between attractions
    • This should take into account hour limit per day
    • This should take into account any other rules I want to impose (no two restaurants in a row, etc.)
    • This should take into account start and end point (default would be hotel/airbnb for both)
  • I need to be able to see commute times based on a chosen mode of transport between wishlist places.
    • Between items in the daily itinerary, I should see something that shows me time it would take to navigate between places
    • When you click on the commute time, some options should appear to be able to change the mode of transportation
